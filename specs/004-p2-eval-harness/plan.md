# Implementation Plan: Evaluation Harness

**Branch**: `004-p2-eval-harness` | **Date**: 2026-04-07 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/004-p2-eval-harness/spec.md`

## Summary

Build an evaluation harness that measures retrieval and answer quality for
the RAG pipeline. Includes a test case corpus (seeded from the About page
sample queries), mechanical retrieval scoring (precision/recall/F1),
LLM-as-judge scoring (accuracy/completeness), baseline comparison with
per-question delta regression detection, and a CLI command to run it all.

## Technical Context

**Language/Version**: Go 1.24+ (same as M1-M3)
**Primary Dependencies**:
- Existing M1-M3 packages (rag, llm, embedder, store, parser, server)
- Anthropic Go SDK (for LLM judge — reuses existing llm package patterns)
- JSON for test case and results storage
**Storage**: JSON files for test cases, baselines, and results
**Testing**: `go test` for eval package, manual validation of judge rubric
**Target Platform**: CLI command, CI pipeline
**Project Type**: CLI tool addition (new `eval` subcommand)
**Performance Goals**: Full eval run under 5 minutes for 6 questions
**Constraints**: Must work with existing pipeline without modifications
**Scale/Scope**: 6 initial test cases, extensible corpus
